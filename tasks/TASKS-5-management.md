    - При ОК: API меняет статус → ACTIVE, публикует посты
  - **"⋯" (три точки)** → выпадающее меню:
    - "🗑️ Удалить" → подтверждение → мягкое удаление
    - "👥 Участники" → если нет Pro подписки → BottomSheet подписки; если есть → список участников
  - **"🔗 Поделиться"** → BottomSheet с двумя табами:
    - Таб "Команды":
      - "Инлайн команда": `@BeastRandomBot [code]` + кнопка "Скопировать"
      - "Команда в боте": `/repost:[code]` + кнопка "Скопировать"
      - Подсказка: "Добавьте своё название в конце для отслеживания статистики"
    - Таб "Ссылки":
      - Поле ввода label + кнопка "Создать"
      - Лимит ссылок по подписке: FREE=3, PLUS=10, PRO=50, BUSINESS=unlimited
      - Список созданных ссылок с кнопками копирования
      - Пустое: "У вас пока нет созданных ссылок"
  - **"📊 Статистика"** → BottomSheet:
    - Если нет подписки: заблюренный контент + стикер + "Статистика доступна для премиум пользователей" + кнопка "Повысить статус"
    - Если есть подписка: реальная статистика (задача 5.2)
- Статус ERROR:
  - Возникает когда: бот не смог опубликовать пост (потерял права), не смог подвести итоги, не смог отправить уведомления
  - На странице: статус "⚠️ Ошибка" + описание ошибки + рекомендация
  - Кнопка "🔄 Повторить" (видна только при ERROR): повторная проверка прав → если восстановлены → retry публикации/завершения → ERROR → ACTIVE/FINISHED
  - Автоматический retry: BullMQ 3 попытки с exponential backoff (1→5→30 мин), после 3 неудач → ERROR фиксируется
  - Раскрывающийся блок "История ошибок" (из GiveawayErrorLog)
- Карточка "Участников: N" обновляется автоматически:
  - MVP: polling каждые 10 секунд `GET /api/giveaways/:id/participant-count`
  - Анимация: число меняется с counter animation (плавно)
  - Для PLUS+: мини-график (sparkline) динамики за 24 часа рядом с числом
- Формат ID розыгрыша: использовать shortCode (8 символов) вместо полного UUID. "Розыгрыш #bf583499" вместо "Розыгрыш #bf583499-9950-4978-a259-804a936f8f3a". Во всех местах: страница управления, участия, уведомления, публикация итогов.
- Real-time обновления (план развития):
  - MVP: polling GET /api/giveaways/:id/participant-count каждые 10 секунд
  - v2 (после MVP): Server-Sent Events (SSE) через `GET /api/giveaways/:id/events`
    - Events: participant_joined (новый участник), participant_count (обновлённый счётчик), status_changed (статус розыгрыша изменился)
    - Fastify: через плагин @fastify/sse или кастомная реализация через response.raw
    - Клиент: EventSource API в Mini App
  - v3 (если нужно): WebSocket через ws или Socket.io
  - Решение о переходе: если polling создаёт > 50% нагрузки на API
